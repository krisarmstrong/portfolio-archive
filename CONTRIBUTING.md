# Contributing to Kris Armstrong GitHub Pages

1. Fork the repository and clone your fork.
2. Create a branch (`git checkout -b feature/your-feature`).
3. Install deps with `npm install` and run `npm run dev`.
4. Keep components under `components/` and styles under `styles/`.
5. Update docs/README when behaviour changes.
6. Commit (`git commit -m "feat: concise summary"`) and push.
7. Open a PR against `main` with screenshots if UI changes.
